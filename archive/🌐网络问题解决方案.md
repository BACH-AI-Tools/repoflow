# 🌐 网络问题解决方案

## 问题分析

### GitHub 连接失败

**错误信息：**
```
Failed to connect to github.com port 443 after 21225 ms: Couldn't connect to server
```

**原因：**
- GitHub 在中国大陆经常被墙或不稳定
- 需要配置代理才能稳定访问

### EMCP/Agent 平台 502 错误

**错误信息：**
```
502 Bad Gateway
```

**原因：**
- 服务器临时不可用或负载过高
- 已添加自动重试机制

---

## 🔧 解决方案

### 方案 1：配置 Git 代理（推荐）

如果你有代理（如 Clash、V2Ray），配置 Git 使用代理：

```powershell
# HTTP 代理（假设代理地址是 127.0.0.1:7890）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# SOCKS5 代理
git config --global http.proxy socks5://127.0.0.1:7890
git config --global https.proxy socks5://127.0.0.1:7890
```

**常见代理端口：**
- Clash: 7890
- V2Ray: 10809
- SSR: 1080

**验证配置：**
```powershell
git config --global --list | findstr proxy
```

**取消代理：**
```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```
